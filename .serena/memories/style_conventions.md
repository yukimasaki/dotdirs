# コードスタイルと規約

## フォーマット設定

### EditorConfig

プロジェクトは `.editorconfig` を使用してコードスタイルを統一しています。

- **改行コード**: LF (Unix スタイル)
- **文字エンコーディング**: UTF-8
- **末尾の改行**: 必須
- **末尾の空白**: 削除

### インデント

- **Shell スクリプト**: スペース 2 つ
- **JavaScript/TypeScript/JSON**: スペース 2 つ
- **YAML**: スペース 2 つ
- **Markdown**: 末尾空白の削除を無効化

## コーディング規約

### Shell スクリプト

- **エラーハンドリング**: `set -e` を使用してエラー時に即座に終了
- **変数**: 小文字のスネークケース（例: `target_dir`）
- **関数**: 小文字のスネークケース（例: `parse_arguments`）
- **定数**: 大文字のスネークケース（例: `DOTDIRS_DIR`）
- **引用符**: 変数展開時は必ず二重引用符を使用
- **モジュール化**: 機能ごとに `libs/` ディレクトリに分割

### ファイル構造

- **メイン処理**: `setup.sh` に配置
- **モジュール**: `libs/` ディレクトリに分割
- **設定**: `config/` ディレクトリにドットファイルを配置
- **テスト**: `__tests__/` ディレクトリに配置

### コメント

- **スクリプトヘッダー**: 各スクリプトファイルの先頭に目的と使用方法を記載
- **セクション区切り**: コメントでセクションを明確に区切る
- **関数コメント**: 複雑な関数には説明コメントを追加

## Prettier 設定

`.prettierrc` ファイルは存在しますが、現在は空です（デフォルト設定を使用）。

## リンター

このプロジェクトでは、明示的なリンター設定はありません。代わりに、ShellSpec テストでコードの動作を検証します。
