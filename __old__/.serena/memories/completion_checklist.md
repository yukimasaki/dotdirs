# タスク完了時のチェックリスト

タスクを完了した際に確認すべき項目です。

## コード変更時

1. **テストの実行**
   ```bash
   shellspec __tests__/
   ```

2. **構文チェック**
   ```bash
   shellspec __tests__/setup_spec.sh --dry-run
   ```

3. **EditorConfig の確認**
   - インデント: スペース 2 つ
   - 改行コード: LF
   - 末尾の空白: 削除済み
   - 末尾の改行: 存在

4. **モジュールの整合性確認**
   - 必要なモジュールが適切に読み込まれているか
   - 関数の依存関係が正しいか

## 新しい機能追加時

1. **ドキュメントの更新**
   - `README.md` に機能の説明を追加
   - 必要に応じて `docs/` に詳細ドキュメントを追加

2. **テストの追加**
   - `__tests__/setup_spec.sh` に新しいテストケースを追加

3. **バックワード互換性の確認**
   - 既存の機能が壊れていないか確認

## ドットファイル追加時

1. **config/ ディレクトリに追加**
   - ルート直下のファイル: `config/.filename`
   - サブディレクトリ内のファイル: `config/.dirname/filename`

2. **テストの確認**
   - 新しいファイルが選択UIに表示されるか確認
   - シンボリックリンクが正しく作成されるか確認

## Git コミット前

1. **テストがすべて成功することを確認**
2. **変更内容を確認 (`git diff`)**
3. **コミットメッセージを適切に記述**
4. **必要に応じて `.gitignore` を確認**
